# 8+16 减脂执行跟踪 MVP 冻结说明

## 1. 产品目标

当前版本只解决一个问题：

**帮助个人用户在一个明确周期内执行 8+16 减脂计划，并持续知道自己今天是否偏离、当前是否接近目标。**

这不是一个完整健康管理系统，也不承担销售型 SaaS 的多用户能力。

---

## 2. 当前版本边界

### 核心保留

1. 目标与周期设置
   - 开始日期
   - 结束日期
   - 初始体重
   - 目标体重
   - 进食窗口开始时间
   - 窗口时长

2. 每日执行记录
   - 记录每次进食
   - 自动判断窗口内 / 窗口外
   - 记录每日体重

3. 执行辅助
   - Telegram 固定三次提醒
   - 首页今日状态
   - 本周提醒摘要

4. 轻量教练复盘
   - 睡眠一键打卡：`<6h / 6-7h / >7h`
   - 运动一键打卡：`20分钟 / 再加10分钟`
   - 系统给出当日重点建议

5. 数据沉淀
   - 本地 JSON
   - Excel 导出
   - 可选同步 Notion

### 当前明确不做

1. 腰围记录
2. 热量 / 营养素计算
3. 复杂运动计划编排
4. 多用户、支付、订阅、销售能力

---

## 3. 首页定义

首页只保留 4 个模块：

1. 目标阶段卡
   - 距离目标还差多少 kg
   - 起始 / 当前 / 目标体重
   - 周期剩余时间
   - 目标进度与节奏判断

2. 今天的任务卡
   - 今天是否偏离
   - 当前窗口
   - 进食次数 / 窗口外次数
   - 主按钮：记录进食
   - 次按钮：记录体重

3. 教练复盘卡
   - 当日重点
   - 一句建议
   - 睡眠 / 运动状态
   - 一键打卡按钮

4. 本周提醒卡
   - 近 7 天偏离摘要
   - 最近一次进食
   - 重点风险提醒
   - 最近记录展开能力

---

## 4. 当前判定规则

1. 今日有窗口外进食：`未达标`
2. 今日有进食记录，且睡眠 < 6h：`未达标`
3. 今日有进食记录，且累计运动 < 20 分钟：`未达标`
4. 今日有进食记录，且无上述偏差：`达标`
5. 今日无进食记录：`未记录`

---

## 5. 当前版本的下一步原则

1. 优先降低录入成本，不优先增加记录维度
2. 优先提高执行率，不优先增加分析复杂度
3. 所有新增功能都要回答一个问题：
   - 它是否直接帮助用户更稳定地完成 8+16 计划？

如果答案不明确，默认不进入 MVP。
