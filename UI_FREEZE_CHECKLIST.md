# UI 冻结清单（8+16 减脂跟踪）

更新时间：2026-04-09

## 1. 冻结范围（本阶段不再改动）

1. 首页信息架构
- 顶部 `Hero`：主标题 + 2 个核心指标（当前体重、目标进度）+ 进度条 + 简要趋势。
- `Today` 卡：状态、窗口、次数、双按钮（记录进食/记录体重）。
- `Activity` 卡：筛选按钮与显示控制分层。

2. 主交互路径
- 记录入口仅保留 `Today` 卡两个按钮。
- 不再恢复悬浮 `+` 入口。
- 动态区默认折叠，按需展开。

3. 语义与色彩
- 达标：绿色。
- 异常：红色。
- 中性/待记录：灰色。
- 主操作：蓝色按钮。

4. 文案基线
- 模块标题中文化（减脂总览、今日状态、动态记录）。
- 目标状态语义统一（如“已达标，超出 x.x kg”）。


## 2. 上线前验收项（必须通过）

1. 视觉一致性
- 同类按钮高度一致（主按钮/次按钮）。
- 同类标签尺寸一致（badge/chip/day-pill）。
- 卡片圆角、阴影、边框节奏统一。

2. 交互一致性
- 同一功能仅一条主路径，不出现重复入口。
- 筛选行为仅影响筛选；显示控制仅影响展开/数量。
- 弹层可正常打开、关闭、保存。

3. 数据语义一致性
- `已减重量` 与 `30天变化`含义不冲突。
- 目标进度文案与颜色一致（达标绿、异常红）。
- “今天/昨天”标签按真实日期计算。

4. 设备可用性（鸿蒙手机）
- 小屏（<=420px）可读，不挤压。
- 高亮环境可读（对比度足够）。
- 触控命中舒适（最小高度 >= 44px）。

5. 回归测试
- `python3 regression_tests.py` 通过。


## 3. 允许迭代项（后续可改，不影响当前上线）

1. 字体与字重微调（仅样式层）。
2. 卡片细节动画与过渡效果。
3. 动态列表的信息密度和排序策略。
4. 深色模式（如需要另开版本评审）。


## 4. 变更控制规则（建议执行）

1. 任何 UI 改动必须满足：
- 不新增重复入口；
- 不改变既有业务判定逻辑；
- 不破坏回归测试。

2. 改动流程：
- 先提交“变更意图（1-2句）”；
- 再改样式或文案；
- 最后跑回归测试并记录结果。
