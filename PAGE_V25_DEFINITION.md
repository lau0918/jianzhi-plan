# V2.5 页面定义稿

更新时间：2026-04-09

## 1. 设计目标

这个页面不是在“汇报数据”，而是在“陪用户完成今天的减肥任务”。

页面需要同时完成 4 件事：

1. 让用户知道自己离目标还有多远。
2. 让用户知道今天下一步该做什么。
3. 让用户知道最近哪里容易失控。
4. 让用户在异常之后还能继续，而不是产生挫败感。


## 2. 页面总结构

首页按以下顺序组织：

1. `目标阶段卡`
2. `今日行动卡`
3. `风险提醒卡`
4. `趋势复盘卡`

信息优先级说明：

- 第一眼看长期目标。
- 第二眼看今天的动作。
- 第三眼看最近风险。
- 最后才看趋势数据。


## 3. 卡片定义

### 3.1 目标阶段卡

目的：让用户知道“我是否正在朝目标前进”。

固定内容：

- 主标题
- 一句副文案
- 进度条
- 两个核心数字：`当前体重`、`目标进度`
- 次级入口：`编辑目标`

不再默认展示：

- 大量并列统计
- 多个同层级重量指标

状态定义：

1. 未设置目标
- 主标题：`先设置减脂目标`
- 副文案：`填写开始日期、初始体重和目标体重后，这里会显示进度`

2. 减重中
- 主标题：`距离目标还差 x.x kg`
- 副文案：`比开始时轻了 x.x kg`

3. 已达标
- 主标题：`你已经达标，接下来重点是稳定`
- 副文案：`当前体重已进入目标范围，继续保持节奏`

4. 反弹或偏离
- 主标题：`最近有反弹，先稳住节奏`
- 副文案：`先回到进食窗口内，不需要追求过度补偿`

视觉要求：

- 这是首页最强视觉卡片。
- 强调目标，不强调细碎报表。
- 绿色只用于“已达标/稳定”。
- 红色只用于“明显偏离”。


### 3.2 今日行动卡

目的：让用户打开页面后，立刻知道“现在该做什么”。

固定内容：

- 标题：`今天的任务`
- 一句行动提示
- 当前窗口信息
- 今日进食次数
- 今日窗口外次数
- 主按钮：`记录进食`
- 次按钮：`记录体重`

不建议加入：

- 假进度条
- 没有真实业务含义的 `0/2` 完成度

状态定义：

1. 未记录
- 标题：`今天还没开始`
- 提示：`先记录第一餐，开始今天的执行`
- 按钮主次保持不变

2. 已记录且正常
- 标题：`今天执行正常`
- 提示：`继续保持，所有进食都在窗口内`

3. 有窗口外进食
- 标题：`今天有偏离`
- 提示：`下一餐回到窗口内就可以，不需要放弃今天`

4. 当天已记录体重但未记录进食
- 标题：`今天还差进食记录`
- 提示：`体重已记录，下一步记录第一餐`

文案原则：

- 使用“下一步行动”语言。
- 避免纯结果型标签。
- 避免强惩罚口吻。


### 3.3 风险提醒卡

目的：优先告诉用户“最近哪里容易失败”。

固定内容：

- 标题：`本周提醒`
- 一句本周总结
- 最近 1-2 条高价值提醒
- 次级入口：`查看全部记录`

内容优先级：

1. 最近异常日
2. 最近未记录日
3. 如果没有异常，再显示“执行稳定”

推荐展示逻辑：

1. 如果近 7 天存在窗口外进食
- 总结：`近7天有 1 天窗口外进食`
- 明细：显示最近异常那一天

2. 如果没有异常但有未记录
- 总结：`近7天没有窗口外进食，但有未记录`
- 明细：显示最近未记录日期

3. 如果近 7 天都正常
- 总结：`近7天执行稳定，没有异常`

不要做成：

- 普通流水账首页
- 所有记录默认平铺


### 3.4 趋势复盘卡

目的：提供辅助复盘，而不是抢主焦点。

建议保留 3 项：

- `连续达标`
- `7天达标率`
- `30天体重变化`

位置：

- 放在风险提醒卡之后
- 视觉权重低于目标卡和今日行动卡

说明：

- 趋势是复盘信息，不是首页第一决策信息。


## 4. 前台展示文案规则

前台不要直接照搬后端判定词。

后端状态到前台文案建议映射：

- `未记录` -> `今天还没开始`
- `达标` -> `今天执行正常`
- `未达标` -> `今天有偏离`
- `异常` -> `需要调整`

原因文案也要去惩罚化：

- 不说：`未达标`
- 更适合：`下一餐回到窗口内就行`

- 不说：`异常`
- 更适合：`今天需要调整一下节奏`

目的：

- 保持真实
- 降低羞耻感
- 提高继续执行的概率


## 5. 现有数据模型支持情况

当前代码已经支持：

1. 目标信息
- `start_date`
- `start_weight`
- `target_weight`
- `current_weight`
- `lost_weight`
- `left_weight`
- `reached`

2. 今日状态
- `status`
- `reason`
- `meal_count`
- `meal_out_window_count`
- `weight`

3. 周期统计
- `7天达标率`
- `7天异常天数`
- `30天体重变化`

4. 按天聚合的记录明细
- 进食
- 体重
- 是否窗口外

当前不建议直接做的内容：

1. `今日完成度 0/2`
原因：当前没有正式的“任务模型”。

2. 复杂行为诊断
原因：目前只有窗口内外、次数、体重，没有足够行为标签。

3. 太多阶段化任务系统
原因：会超过当前数据和产品复杂度。


## 6. V2.5 的落地原则

实现时遵循以下规则：

1. 先改结构，再改视觉
- 先把卡片职责理顺
- 再做细节美化

2. 首页只能有一个最强主动作
- 固定为 `记录进食`

3. 风险优先于日志
- 用户需要的是提醒，不是流水账

4. 用语必须支持坚持
- 不能让用户一打开就感到“我又失败了”

5. 不新增虚假指标
- 所有数字和状态都必须能被现有数据支撑


## 7. 推荐实施顺序

第一步：
- `今日状态` 改为 `今天的任务`
- 重写标题和引导文案

第二步：
- `减脂动态` 改为 `本周提醒`
- 默认显示风险摘要，不默认显示日志列表

第三步：
- 趋势卡从目标卡中降级或下移

第四步：
- 再做视觉精修


## 8. 一句话版本

V2.5 的首页不是在告诉用户“你做得怎么样”，而是在帮助用户“今天继续做下去”。
