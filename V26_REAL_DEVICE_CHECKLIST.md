# V2.6 真机走查清单（鸿蒙优先）

更新时间：2026-04-09

目的：验证“周期内达标”主线是否在真实设备上可理解、可执行、可持续。

## 1. 走查范围

本清单只验证首页核心链路：

1. `目标阶段卡`（体重进度 + 时间进度 + 节奏判定）
2. `今天的任务`（行动提示是否明确）
3. `本周提醒`（风险是否优先、是否可展开全部记录）
4. `趋势复盘`（作为次级信息，不抢主焦点）


## 2. 环境准备

1. 启动服务  
`python3 mobile_server.py --host 0.0.0.0 --port 8000`

2. 手机与电脑同局域网，打开：  
`http://<电脑IP>:8000`

3. 若页面缓存旧资源，先强制刷新（清缓存后刷新）。

4. 推荐用场景脚本快速切换数据（可选）：  
`python3 v26_scenario_builder.py --scenario A`  
`python3 v26_scenario_builder.py --scenario B`  
`python3 v26_scenario_builder.py --scenario C`  
`python3 v26_scenario_builder.py --scenario D`

说明：
- 默认写入 `fasting_data.json`
- 若文件已存在会自动备份
- 也可指定输出路径（不覆盖正式数据）：  
`python3 v26_scenario_builder.py --scenario A --output /tmp/v26_A.json --no-backup`


## 3. 关键状态用例

## 用例 A：周期进行中 + 正常节奏

目标：确认页面能同时表达“还差多少kg”和“还剩多少天”。

准备数据（CLI）：
1. `python3 fasting_tracker.py set-goal --start-date 2026-04-01 --end-date 2026-06-01 --start-weight 90 --target-weight 80`
2. `python3 fasting_tracker.py set-window --start 07:00 --hours 8`
3. 录入至少2条体重记录（近7天内），让系统可计算本周实际变化。

页面预期：
1. 目标卡显示周期区间：`2026-04-01 ~ 2026-06-01`
2. 显示 `第x/y天` 与 `剩余x天`
3. 目标卡存在两条进度：体重进度、时间进度
4. 节奏判定显示为：`按计划：当前进度正常` 或 `略快于周期节奏`


## 用例 B：冲刺期（剩余 <= 14 天）

目标：确认系统在临近周期结束时主动提醒，不靠用户自己计算。

准备数据：
1. 将 `end_date` 调整到距离今天 14 天以内（可在设置页保存）。

页面预期：
1. `今天的任务`出现冲刺提示：`冲刺期：还剩 x 天`
2. 主按钮仍保持唯一主动作（记录进食）
3. 本周提醒仍优先展示风险摘要，不退化成流水账


## 用例 C：周期结束 + 未达标

目标：确认结束态不会“沉默失败”，而是给下一步动作。

准备数据：
1. 设置 `end_date` 早于今天
2. 保持 `current_weight > target_weight`

页面预期：
1. 目标卡显示周期结束信息（如 `周期已结束 +x天`）
2. 节奏判定为 `周期已结束且未达标`（或等价语义）
3. `今天的任务`给出下一步：`建议开启下一周期继续执行`
4. 不出现“已达标/进行顺利”等错误正向文案


## 用例 D：周期结束 + 已达标

目标：确认达标后进入维持语义，而不是继续施压。

准备数据：
1. 设置 `end_date` 早于今天
2. 保持 `current_weight <= target_weight`

页面预期：
1. 目标卡主文案为“已达标/稳定”方向
2. 不再显示“落后节奏”类负向提示
3. 今日任务保持可记录，但文案偏“稳定执行”


## 4. 交互检查项

1. 本周提醒的 `查看全部记录` 按钮可展开/收起
2. 展开后 `全部/异常/达标` 筛选工作正常
3. `查看全部` 与 `只看最近5条` 切换正常
4. `记录第一餐`、`记录体重` 弹层打开/保存/关闭无异常


## 5. 可读性检查项（鸿蒙）

1. 强光环境：深色进度区文字是否仍可读
2. 小屏（窄宽度）：
- 目标卡主标题是否换行自然
- 周期信息是否挤压重叠
3. 触控：
- 主按钮最少 44px 高，连续点击不误触
- 记录展开按钮可点击区域足够


## 6. 通过标准

必须全部满足：

1. 四个关键状态用例均通过
2. 交互检查项全部通过
3. 可读性检查项无阻断问题
4. 回归测试通过：`python3 regression_tests.py`
